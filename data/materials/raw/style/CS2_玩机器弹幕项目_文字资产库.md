# CS2 玩机器直播间弹幕项目：文字资产库

> 版本：v0.1  
> 更新日期：2026-05-02  
> 用途：作为项目中的基础文案、命名、梗解释、弹幕生成、数据标注和内容策划资产。  
> 说明：队伍阵容变化较快，本文件中的“公开阵容”应作为快照使用；正式上线前建议从 HLTV / Liquipedia 再刷新一次。

---

## 1. 项目定位

本项目围绕 CS2 中文赛事观看语境，重点提炼“玩机器 / 6657”直播间中的解说风格、弹幕风格、战队与选手称呼、二创黑话、烂梗模板和可复用语料结构。

核心目标：

1. 建立一套可复用的文字资产库。
2. 支持弹幕生成、梗解释、选手 / 队伍别称映射、直播间风格模拟。
3. 把零散弹幕语感整理成结构化数据，方便后续用于产品、脚本、机器人、内容生成或数据标注。

## 6. 玩机器解说风格资产

### 6.1 总体风格

玩机器直播间的文本语境不是单纯赛事解说，而是“赛事解说 + 主播反应 + 弹幕共创 + 烂梗复读 + 黑话二创”的混合体。其核心不是严肃战术分析，而是把比赛过程转化成可复读、可复制、可二创的娱乐文本。

### 6.2 解说语言特征

| 维度 | 特征 | 项目可复用写法 |
|---|---|---|
| 语速与节奏 | 快速反应，跟随击杀和残局瞬间起伏 | 短句、高频感叹、连续反问。 |
| 情绪表达 | 红温、崩溃、惊讶、阴阳怪气并存 | “这也能输？”“不是哥们？”“这波什么道理？” |
| 判断方式 | 常用结果导向、画面导向、节目效果导向 | “这波打完就知道谁在裸泳”。 |
| 选手叙事 | 把选手变成稳定人设 | 大爹、老登、天才少年、研发对象、战犯。 |
| 弹幕互动 | 弹幕不是背景，而是内容共同作者 | 主播一句话引出弹幕复读，弹幕再反向塑造梗。 |
| 反差幽默 | 用夸张比喻解释离谱局面 | “这不是残局，这是科研现场”。 |

### 6.3 可复用解说句式

```text
不是，这波他凭什么敢这样打？
这枪一出，研发经费直接到账。
这不是默认，这是默认等死。
这残局打完，弹幕已经开始写论文了。
这队现在最大的问题不是枪法，是大家都觉得自己有想法。
他这一死，整个战术瞬间从科研变玄学。
你别说，这波还真给他圆回来了。
这把不是输了，是被弹幕提前判了。
```

## 7. 6657 弹幕风格资产

### 7.1 弹幕总体风格

6657 弹幕的核心是“高密度二创 + 复读传播 + 低成本变体 + 强攻击性柔化”。弹幕通常不追求完整叙事，而追求瞬间识别、快速复制、群体同步和节目效果。

### 7.2 弹幕常见机制

| 机制 | 说明 | 示例方向 |
|---|---|---|
| 复读 | 一个句式被大量复制，形成屏幕效果 | “没意思，不看了”“研发结束”。 |
| 反讽 | 表面夸奖，实际阴阳 | “太有想法了，建议继续研发”。 |
| 柔化攻击 | 用符号或谐音降低攻击性 | “火÷”“洞出”。 |
| 科研化 | 把比赛失误、选手博弈说成研发 | “影响研发”“研发成果”“实验失败”。 |
| 表情符号化 | 用 emoji 代替态度和身份 | 复活节岛石像表示沉默、抽象、无语或“洞出”语境。 |
| 选手符号化 | 选手 ID 被压缩为单字或黑称 | flameZ → 火 / 火÷；donk → 东 / donk。 |
| 结论先行 | 弹幕先判定，再看比赛 | “结束了”“没意思，不看了”。 |

### 7.3 高频弹幕模板

```text
[选手] 今天在影响研发。
[选手A] 杀死 [选手B] 后，研发结束。
复活节岛石像：[单字选手名]÷
没意思，不看了。
不是哥们，这也能研发？
这波实验数据很充分。
建议把这回合送去同行评审。
研发失败，申请经费重开。
[队伍] 这把不是在打比赛，是在交课题。
```

## 8. “研发梗”专题整理

### 8.1 梗的核心含义

“研发梗”把 CS2 比赛中的非常规打法、离谱失误、强行操作、战术尝试、选手互杀、残局翻盘等情况，统一包装成“科研 / 研发 / 实验”。它的优势是解释空间大：打得好可以说“研发成功”，打得烂可以说“实验失败”，选手死亡可以说“影响研发”或“研发结束”。

### 8.2 flameZ / 火÷ 相关用法

用户已明确的项目设定：

- “÷”来自对过激词语的二创柔化，常用于把攻击性称呼压缩成符号化表达。
- flameZ 可被压缩为“火”。
- “火÷”是围绕 flameZ 的戏谑表达。
- “洞出 / 火÷”可与复活节岛石像 emoji 共同出现。
- 示例：“flameZ 杀死 donk 后，研发结束。”
- 示例：“影响研发的时候，还会出现：复活节岛石像：[单字，影响研发的选手]÷；没意思，不看了。”

### 8.3 研发梗模板库

| 场景 | 模板 | 示例 |
|---|---|---|
| flameZ 击杀 donk | `[flameZ / 火] 杀死 [donk / 东] 后，研发结束` | `火杀死东后，研发结束。` |
| 某选手失误 | `[选手] 影响研发` | `火÷又在影响研发。` |
| 某队连续白给 | `[队伍] 研发经费不足` | `小蜜蜂这把研发经费见底了。` |
| 离谱战术成功 | `研发成功，建议量产` | `这 B 点夹击研发成功，建议量产。` |
| 离谱战术失败 | `实验失败，申请重开` | `这个强摸烟实验失败，申请重开。` |
| 关键人物阵亡 | `[选手] 一死，课题没了` | `donk 一死，课题没了。` |
| 弹幕无语 | `🗿：[单字]÷` | `🗿：火÷` |
| 比赛变无聊 | `没意思，不看了` | `研发结束，没意思，不看了。` |

### 8.4 安全改写建议

原始弹幕可能含有攻击性词汇。项目公开展示时，建议采用以下替换策略：

| 原始强攻击表达 | 项目内柔化表达 | 说明 |
|---|---|---|
| 直接辱骂 | `X÷` | 保留弹幕风格，降低直白攻击性。 |
| 人身攻击 | `影响研发` | 转化为玩法评价。 |
| 过激嘲讽 | `没意思，不看了` | 用复读梗替代攻击。 |
| 阴阳选手 | `实验样本 / 课题对象` | 转化为科研叙事。 |

## 9. 弹幕生成规则

### 9.1 输入字段建议

```json
{
  "match": "Vitality vs Spirit",
  "team_a": "Vitality",
  "team_b": "Spirit",
  "player_focus": "flameZ",
  "event": "flameZ kills donk",
  "round_context": "关键回合",
  "tone": "6657研发梗",
  "toxicity_level": "softened",
  "output_type": "danmaku"
}
```

### 9.2 输出风格规则

1. 尽量短，适合弹幕横向滚动。
2. 优先使用选手 ID、单字别称、队伍别称。
3. 避免长解释；梗解释另做注释层。
4. 可使用符号、emoji、复读句式。
5. 攻击性词汇默认柔化。

### 9.3 输出示例

```text
火杀死东后，研发结束。
🗿：火÷
没意思，不看了。
这波不是击杀，是课题终止。
donk 倒了，实验组散会。
小蜜蜂今天研发经费很足。
Falcons 这阵容不是打比赛，是烧钱做实验。
老登开会，小孩做题。
```

## 12. 可直接放入项目的说明文案

### 12.1 项目简介

```text
这是一个面向 CS2 中文观赛社区的弹幕与烂梗文字资产库，重点整理玩机器直播间语境中的战队别称、选手黑话、解说风格、弹幕模板和二创梗。项目用于支持弹幕生成、梗解释、文本标注和社区内容分析。
```

### 12.2 研发梗说明

```text
“研发梗”是玩机器直播间弹幕中常见的一种科研化叙事方式。弹幕会把选手的非常规操作、离谱失误、关键击杀或战术尝试称为“研发”“实验”“课题”。当某个选手击杀关键人物，或让比赛走向发生变化时，弹幕会说“研发结束”“影响研发”“实验失败”等。
```

### 12.3 安全展示说明

```text
本项目保留弹幕社区的语言风格，但会对强攻击性、人身攻击和不适合公开展示的表达进行符号化或柔化处理。例如使用“X÷”“影响研发”“没意思，不看了”等形式保留语感，同时降低直白攻击性。
```
