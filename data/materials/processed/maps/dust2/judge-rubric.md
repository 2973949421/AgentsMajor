# Dust2 裁判规程资产

## 1. 文档定位

这份文档是 `processed/maps/dust2/` 下的独立裁判资产。

它负责把 `Dust2` 上 judge 到底看什么、怎么判、哪些证据能用、哪些因素禁止介入，单独提升为一等规则文档。

本稿默认服务：

```text
Dust2 机会定位图
Falcon-7B vs VitaLLMty 单图 / 定制 BO3 校准
后续同类“机会定位图”变体
```

## 2. Dust2 上 judge 的核心任务

Dust2 的 judge 不是在判“谁说得更像赢了”，而是在判：

```text
哪一方在当前回合更有效地攻击了对手的机会判断漏洞，
或更有效地守住了己方机会判断的核心成立点。
```

Dust2 的总命题固定为：

```text
这个项目到底值不值得做？
```

因此 judge 在 Dust2 上只能围绕“机会是否真实”裁决，不允许把完整商业闭环、完整增长体系、完整规模化治理拿来替代 Dust2 的主判轴。

## 3. Dust2 judge 的职责边界

### judge 必须负责

```text
判断当前回合谁更贴合 Dust2 命题。
判断攻方是否真正打中了对手的机会缺口。
判断守方是否真正守住了己方的核心成立点。
判断当前胜负理由能否累计成整图胜势。
在整场 BO3 结束后总结双方机会判断的强弱与修正方向。
```

### judge 不负责

```text
替任一队伍补完其本来没有讲清楚的方案。
因为队名、名气、先后顺序、比分领先而自动倾向某队。
把 Dust2 判成商业路演、增长辩论或工程答辩。
在没有 coach 介入窗口时主动假设 coach 已经修正过方案。
```

## 4. Dust2 judge 的输入证据

Dust2 judge 允许消费的输入证据，按优先级从高到低分为三层。

### A 层：回合直接证据

```text
当前地图命题：机会定位图
当前回合子命题：R1-R6 / OT1-OT3
当前攻守身份
当前主攻区 / 主守区
双方本回合 team plan
双方 10 名 player action
当前回合关键事件与摘要
当前比分、经济状态、buy type
```

### B 层：本图上下文证据

```text
两队本图前已携带的初始方案包摘录
前几回合已暴露出的机会缺口
最近 2-3 回合的图内趋势
暂停窗口内 coach 的明确修正意见
```

### C 层：弱上下文

```text
队伍人设
角色标签
历史 hooks
转播包装语
```

### 禁止作为裁决依据的输入

```text
哪支队伍名字更强。
哪支队伍更像现实世界名队。
teamA / teamB 的固定顺序。
谁当前比分领先。
哪一方写得更长、更华丽。
与 Dust2 主命题无关的商业、增长、品牌宏大叙事。
```

## 5. Dust2 judge 的裁决主轴

| 维度 | Dust2 要看什么 | 常见进攻打法 | 常见防守任务 |
|---|---|---|---|
| `ICP clarity` | 第一用户 / buyer 是否清楚、聚焦、可信。 | 打宽用户、假 buyer、伪 ICP。 | 守住核心人群和优先用户。 |
| `pain severity` | 痛点是否真实、尖锐、持续。 | 打伪需求、弱痛点、轻问题。 | 守住“这个问题真的很痛”。 |
| `scenario specificity` | 场景是否具体、真实、高频。 | 打低频、空场景、伪触发。 | 守住关键使用场景和触发时机。 |
| `wedge sharpness` | 切口是否足够锋利，是否真能先打穿。 | 打大词、泛方案、钝切口。 | 守住最小可打穿入口。 |
| `timing credibility` | 为什么是现在，而不是以后再说。 | 打错误时机、假窗口、无 urgency。 | 守住 why now 的成立依据。 |

judge 必须优先围绕这 5 项写 reason，不得漂到 Dust2 命题之外。

## 6. Dust2 judge 的回合判定流程

### 第一步：确认本回合命题

```text
当前是 R1-R6 的哪个子命题，或 OT1-OT3 的哪个压缩子命题。
```

### 第二步：确认攻守任务

```text
谁在攻。
谁在守。
攻方主攻的是哪个机会缺口。
守方主守的是哪个核心成立点。
```

### 第三步：检查攻方是否打中

judge 需要判断攻方是否真正打中了以下任一类缺口：

```text
伪需求
宽用户
弱痛点
低频场景
伪切口
错误时机
```

### 第四步：检查守方是否守住

judge 需要判断守方是否真正守住以下任一类核心点：

```text
核心用户
核心痛点
核心场景
最锋利切口
why now 依据
```

### 第五步：回到 Dust2 五维主轴

judge 必须至少明确比较其中 3 项以上主轴，且必须包含当前回合子命题最相关的那一项。

### 第六步：形成胜负结论

```text
winnerTeamId
loserTeamId
margin
confidence
reason
```

## 7. Dust2 的回合评分口径

| 项目 | 判断问题 | 倾向性 |
|---|---|---|
| 命题贴合度 | 这段攻防有没有紧贴当前 Dust2 子命题。 | 不贴题者先劣。 |
| 攻方打击质量 | 攻方是否打中了对手真实缺口。 | 打中者占优。 |
| 守方保点质量 | 守方是否守住最关键成立点。 | 守住者占优。 |
| 证据硬度 | 论证是否具体、可落点、可回指。 | 空泛者劣。 |
| Dust2 五维主轴 | 谁在当前最相关维度上更成立。 | 更成立者占优。 |

### margin 建议口径

```text
1-2：微弱优势，本回合双方都还成立，但一方更贴题。
3-4：明确优势，一方打穿了对手关键点或稳稳守住本方关键点。
5：压制优势，一方在当前子命题上形成显著碾压。
```

### confidence 建议口径

```text
0.50-0.64：证据偏薄，但已有倾向。
0.65-0.79：证据较清楚，回合可判。
0.80-0.90：证据明确，主轴成立清楚。
0.90+：仅在当前回合出现明显打穿或明显崩塌时使用。
```

## 8. Dust2 的 reason 硬约束

每回合 reason 必须同时覆盖：

```text
当前子命题是什么。
攻方打了对手哪个机会缺口。
守方守了或没守住哪个核心成立点。
为什么这个结论符合 Dust2 的机会定位命题。
```

### 明确禁止的 reason

```text
只说“整体更完整”。
只说“更有说服力”。
只说“更像赢家”。
只说“商业上更大”。
只说“增长潜力更强”。
```

## 9. Dust2 的偏置防护

judge 在 Dust2 上必须显式防以下偏置：

```text
顺序偏置。
名气偏置。
领先偏置。
文风偏置。
大词偏置。
```

### 交替阅读原则

如果系统允许控制输入顺序，Dust2 judge 应交替改变两队在提示词中的先后顺序。

### 同比分支独立判断原则

judge 不得因为某队已经连赢数局，就自动延续其连胜。每回合必须重新回到当前子命题和当前证据。

## 10. Dust2 的连胜复核规则

当出现以下情况时，应触发 `judge_review`：

```text
同队连续赢 3 局及以上。
本回合 confidence 仍然较高。
当前 reason 没有充分解释败方为什么失守。
当前 reason 没有明确回指 Dust2 子命题。
```

### review 后的处理

```text
如果复核后仍是同一胜方，且 reason 完整，则接受。
如果复核 reason 仍不解释败方为何失守，则该回合判定失败，不提交该回合事实。
```

## 11. coach 介入窗口下的 judge 规则

judge 只能在以下窗口消费 coach 影响：

```text
战术暂停
整场 BO3 赛后复盘
```

在普通回合中，judge 不得假设 coach 已经临场帮队伍补完新总纲。

## 12. Dust2 的输出契约

### 回合级输出

```text
winnerTeamId
loserTeamId
margin
confidence
reason
```

### 建议追加字段

```text
currentSubTheme
attackingTeamId
defendingTeamId
attackedOpportunityGap
defendedCoreProposition
mainAttackZone
mainDefenseZone
decisiveEvidence
reviewTriggered
```

### 赛后输出

```text
哪支队伍的机会判断整体更成立。
对方主要输在什么缺口。
双方各自最需要修正的 1-2 个点。
哪些结论应带去下一场。
```
